## Decisions Made
- **Deferred FK ordering:** `overtime.holiday_id` is a plain `text` column in 00031; the FK to `holidays(id)` is added via `ALTER TABLE overtime ADD CONSTRAINT overtime_holiday_id_fkey` at the end of 00032 (goose runs 00031 before 00032). Down drops the constraint before dropping `holidays`.
- **Decision trail as a table:** `overtime_approvals` is a separate `bigserial`-PK table (mirrors `leave_approvals`/`placement_history`) rather than denormalized columns — avoids touching `ids.go` and matches the openapi `approvals[]` array shape.
- **`holiday_date` column name:** avoids the SQL `date` reserved-word friction; 09-02 maps it to the openapi `date` field at the DTO boundary.
- **Reference multiplier stored only (INV-2):** `numeric(4,2)` column → `*float64` domain field, with NO monetary method anywhere.
- **overtime_rules reused:** no rule CRUD added; the E2/Phase-3 `db/queries/org/overtime_rules.sql` is the rule source for OT_BELOW_MIN/day_type in 09-02.

### Tables / columns / indexes

**`overtime`** (migration 00031) — id `text` PK DEFAULT `'SWP-OT-'||swp_next_id('OT')`. Columns:
`employee_id` (NOT NULL → employees), `company_id` (nullable → client_companies, DENORMALIZED for leader scope), `placement_id` (NOT NULL → placements), `attendance_id` (nullable → attendance, non-null only for AUTO_DETECTED), `service_line_id` (nullable text, denormalized for rule lookup), `work_date` (date NOT NULL), `planned_start_time`/`planned_end_time`/`actual_start_time`/`actual_end_time` (nullable text HH:MM), `cross_midnight` (bool), `source` (CHECK REQUESTED/AUTO_DETECTED/WORKED_WITHOUT_REQUEST), `status` (DEFAULT `PENDING_AGENT_CONFIRM`, CHECK 6 states), `day_type` (CHECK WORKDAY/RESTDAY/HOLIDAY), `worked_minutes`/`counted_minutes`/`min_minutes_threshold` (int, threshold DEFAULT 30), `skipped_too_short` (bool), `reference_multiplier` (numeric(4,2) nullable — STORED only, INV-2), `overtime_rule_id` (nullable → overtime_rules), `holiday_id` (nullable → holidays, FK added in 00032), `flagged_no_preapproval` (bool), `reason` (text), `created_by`, `created_at`/`updated_at`/`deleted_at`.
Indexes: `overtime_company_status_idx (company_id,status) WHERE deleted_at IS NULL`, `overtime_employee_idx (employee_id) …`, `overtime_holiday_idx (holiday_id) …`.

**`overtime_approvals`** (migration 00031) — `id bigserial` PK, `overtime_id` (NOT NULL → overtime), `level int CHECK (1,2)`, `decision text CHECK (APPROVED/REJECTED/OVERRIDE_APPROVED)`, `approver_id`/`approver_name`/`reason` (nullable text), `decided_at timestamptz`. Index `overtime_approvals_ot_idx (overtime_id)`.

**`holidays`** (migration 00032) — id `text` PK DEFAULT `'SWP-HOL-'||swp_next_id('HOL')`. Columns: `name` (NOT NULL, maxLength enforced in service), `holiday_date` (date NOT NULL — maps to openapi `date`), `category` (CHECK NATIONAL/REGIONAL/CUSTOM), `recurring` (bool DEFAULT false), `applicable_service_lines text[] DEFAULT '{}'` (empty = global), `created_at`/`updated_at`/`deleted_at`. Unique index `holidays_date_category_uq (holiday_date,category) WHERE deleted_at IS NULL` (HOLIDAY_DATE_CLASH 23505 backstop).

### Querier method signatures (key ones)

- `ListOvertime(ctx, ListOvertimeParams) ([]ListOvertimeRow, error)` — keyset cursor DESC; filter fields: `EmployeeID/CompanyID/Status/DayType/Source *string`, `StatusIn []string`, `WorkFrom/WorkTo pgtype.Date`, `FlaggedNoPreapproval *bool`, `CursorCreatedAt *time.Time`, `CursorID *string`, `Lim int32`. Rows include `EmployeeName *string`, `CompanyName *string` (LEFT JOIN nullable).
- `GetOvertime(ctx, id string) (GetOvertimeRow, error)` — joined names; pgx.ErrNoRows → map to domain.ErrNotFound in repo.
- `GetOvertimeForUpdate(ctx, id string) (GetOvertimeForUpdateRow, error)` — `FOR UPDATE` lock, NO joins; use inside the InTx transition.
- `UpdateOvertimeStatus(ctx, UpdateOvertimeStatusParams{ID, Status}) (UpdateOvertimeStatusRow, error)` — RETURNING-or-(no row → 409) transition writer.
- `InsertOvertime(ctx, InsertOvertimeParams) (InsertOvertimeRow, error)` — `ID *string` narg (COALESCE to DEFAULT), `ON CONFLICT (id) DO NOTHING`; seed/HR-on-behalf path.
- `ListOvertimeApprovals(ctx, overtimeID string) ([]OvertimeApproval, error)` — chronological; `InsertOvertimeApproval(ctx, InsertOvertimeApprovalParams{OvertimeID string, Level int32, Decision string, ApproverID/ApproverName/Reason *string}) (…, error)`.
- **Holidays:** `ListHolidays(ListHolidaysParams{Category/ServiceLineID *string, Year *int32, CursorDate pgtype.Date, CursorID *string, Lim int32})` — cursor ASC on (holiday_date,id) using `>`. `GetHoliday(id)`, `GetHolidayByDateCategory(holiday_date, category)` (CLASH pre-check), `GetHolidayForDate(holiday_date)` (day_type classification, NATIONAL-priority). `InsertHoliday` (`ID *string` narg + ON CONFLICT DO NOTHING), `UpdateHoliday` (COALESCE partial — all update fields are nargs: `Name/Category *string, HolidayDate pgtype.Date, Recurring *bool, ApplicableServiceLines []string`), `SoftDeleteHoliday(id) → id`, `CountOvertimeUsingHoliday(holidayID *string) (int64, error)` (HOLIDAY_IN_USE guard + `in_use_by_overtime` flag).

### sqlc type quirks (repo must convert at the boundary, mirrors Phase-7/8)
- `work_date` / `holiday_date` → `pgtype.Date` (convert ↔ `time.Time`: `pgtype.Date{Time: t, Valid: true}` on write, `.Time` on read).
- `worked_minutes`/`counted_minutes`/`min_minutes_threshold` → `int32`; `level` (approval) → `int32`. Domain uses `int` — cast.
- `reference_multiplier` → `pgtype.Numeric` (nullable). Convert ↔ `*float64` in the repo (Numeric `Valid` flag; use `.Float64Value()` / set via a `big.Int`+`Exp` or `Scan` from a string). Domain holds `*float64`.
- `applicable_service_lines` → `[]string` directly (pgx text[] maps natively; `emit_empty_slices: true` so empty = `[]string{}` not nil).
- `WorkedMinutes *int32` appears on the attendance model row (pre-existing) — the **overtime** row's `WorkedMinutes` is non-pointer `int32` (NOT NULL DEFAULT 0). Don't confuse the two.
- `employee_name`/`company_name` on list/get rows are `*string` (LEFT JOIN nullable) — map to domain `*string`.
- `CountOvertimeUsingHoliday` returns `int64`.
- `attendance_id`/`service_line_id`/`overtime_rule_id`/`holiday_id`/`reason`/`created_by`/`planned_*`/`actual_*` → `*string` (nullable).

### day_type / holiday / OT_BELOW_MIN query signatures (for 09-02 classification + rule enforcement)
- **day_type classification:** call `GetHolidayForDate(work_date)` → if a row returns, the work_date is a HOLIDAY tier (NATIONAL-priority pick). Combine with the E4 schedule lookup (scheduled shift → WORKDAY else RESTDAY) via `overtime.TierPrecedence(scheduleTier, holidayTier)` (HOLIDAY > RESTDAY > WORKDAY). Store the resolved `day_type` on the record at seed/record time (CONTEXT discretion).
- **OT_BELOW_MIN:** look up the applicable `overtime_rules` row (reuse the E2/Phase-3 `db/queries/org/overtime_rules.sql` Querier — match by `day_type` + optional `service_line_id`; line-scoped rule wins over global per OR-2). Compare `counted_minutes` (= `overtime.CountedFromWorked()` = floor(worked/30)*30) against the rule's `min_minutes` (locked 30); below → set `skipped_too_short=true` + return `apperr.Rule("OT_BELOW_MIN")` 422 with field errors (INV-5).
- **HOLIDAY_IN_USE:** before `SoftDeleteHoliday`, call `CountOvertimeUsingHoliday(holidayID)` — `> 0` → 409 `HOLIDAY_IN_USE`. The same count drives the `in_use_by_overtime` DTO flag on GET/list holiday responses.
- **HOLIDAY_DATE_CLASH:** before `InsertHoliday`, `GetHolidayByDateCategory(date, category)` pre-check; backstop on the 23505 from `holidays_date_category_uq` → 409 `HOLIDAY_DATE_CLASH`.

### Enums (domain/overtime — match openapi byte-for-byte)
- `OvertimeStatus`: PENDING_AGENT_CONFIRM, PENDING_L1, PENDING_HR, APPROVED, REJECTED, WITHDRAWN.
- `OvertimeSource`: REQUESTED, AUTO_DETECTED, WORKED_WITHOUT_REQUEST.
- `OvertimeTier` (day_type): WORKDAY, RESTDAY, HOLIDAY (`TierPrecedence(a,b)` returns the higher).
- `HolidayCategory`: NATIONAL, REGIONAL, CUSTOM.
- Approval decision strings (overtime_approvals.decision): APPROVED, REJECTED, OVERRIDE_APPROVED; `level` 1=L1/leader, 2=HR final.

### Workflow state machine reminder (09-02 to implement)
PENDING_AGENT_CONFIRM →(`:confirm`)→ PENDING_L1 →(`:approve-l1`, leader, own company)→ PENDING_HR →(`:approve-final`, HR)→ APPROVED. `:reject`→REJECTED, `:withdraw`→WITHDRAWN. Acting on a terminal/wrong-state record → 409. Each transition writes an `overtime_approvals` row in-tx + audit + notify stub. Use `GetOvertimeForUpdate` lock → guard transition → `UpdateOvertimeStatus` (RETURNING-or-409) → `InsertOvertimeApproval`, all in one `db.TxManager.InTx`.

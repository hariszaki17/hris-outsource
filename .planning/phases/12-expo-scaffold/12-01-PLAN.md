existing pnpm/Turborepo monorepo, reaching the v1.1 done-definition: the app boots, resolves
the three surface-agnostic shared packages, styles via NativeWind driven by the shared design

This is foundation work: every later agent/shift-leader screen builds on it. Get the wiring
right (Metro monorepo resolution, token-driven theme, native deps locked up front) so future
milestones never re-litigate tooling and rarely force a store reinstall.
</objective>

<context>
- Monorepo: pnpm@9.6.0, Node 22, Turborepo 2.3.3, Biome 1.9.4, TypeScript 5.7.3.
- Workspace globs already include `apps/*` (frontend/pnpm-workspace.yaml).
- Shared packages export TS source directly (no build step):

  - `@swp/shared` â€” `./ids`, `./datetime` (Asia/Jakarta), `./i18n`.

- Expo SDK: latest stable (expo@^56). Use `npx create-expo-app` template, then graft into the workspace (do not let it create a nested lockfile or node_modules that fight pnpm).
</context>

<approach>
Scaffold by hand-grafting onto the workspace rather than letting create-expo-app own the tree
(it assumes npm/yarn + hoisted node_modules). Concretely:

1. **Generate Expo Router app structure** into apps/mobile: `app/_layout.tsx`, `app/index.tsx`,
   `app.json`, `tsconfig.json` (extends expo/tsconfig.base, strict), `babel.config.js`
   (babel-preset-expo + nativewind/babel), entry via `expo-router/entry`.
2. **package.json**: name `@swp/mobile`; scripts `start`/`android`/`ios`/`typecheck`
   (`tsc --noEmit`)/`lint` (delegated to root biome or local). Deps: expo, expo-router,
   react/react-native at Expo-pinned versions, nativewind + tailwindcss, react-native-reanimated
   + react-native-safe-area-context + react-native-screens, expo-location, expo-notifications,
   expo-image-picker, expo-updates, expo-constants, expo-linking, expo-status-bar. Workspace deps:
   `@swp/api-client`, `@swp/shared`, `@swp/design-tokens` at `workspace:*`. **No** `@swp/ui`.
3. **Metro** (`metro.config.js`): start from `expo/metro-config`, wrap with `nativewind/metro`
   (input `global.css`), set `watchFolders=[workspaceRoot]`, `resolver.nodeModulesPaths=
   [projectRoot/node_modules, workspaceRoot/node_modules]`, `resolver.disableHierarchicalLookup=true`,
   `resolver.unstable_enableSymlinks=true`. This is the canonical Expo+pnpm-monorepo recipe.
4. **NativeWind theme from tokens**: `src/theme/tokens.ts` imports `@swp/design-tokens` and
   reshapes color/space/type into a Tailwind theme object; `tailwind.config.js` spreads it into
   `theme.extend` and sets `content` to `app/**` + `src/**`; `global.css` has the three
   @tailwind directives; `nativewind-env.d.ts` references nativewind types. App code uses
   className tokens (e.g. `text-primary`), never raw hex.
5. **Native deps**: add all four to `app.json` `plugins` with permission/usage strings
   (location when-in-use + background note, notifications, photo library). `expo-updates`
   config (`runtimeVersion.policy`, updates.url placeholder) + `src/lib/update-gate.ts`
   stub: on launch check `Updates.checkForUpdateAsync()`; documented TODO that a future
   backend `GET /app/version-policy` returning `min_supported_version` will drive a hard
   store-update gate (not implemented this milestone).
6. **Smoke screen** (`app/index.tsx`): import a branded-ID helper from `@swp/shared/ids`, a
   date formatter from `@swp/shared/datetime`, a color from `@swp/design-tokens`, and reference
   a query-client/hook type from `@swp/api-client` (type-only is fine) â€” render them so the
   bundle proves all three resolve. Use NativeWind className styling.
7. **Tooling**: tsconfig strict; ensure `turbo run typecheck` picks up the mobile `typecheck`
   script; extend root Biome to include apps/mobile (or local biome.json extending ../../biome.json
   with RN globals). Run `pnpm install`, `pnpm --filter @swp/mobile typecheck`, `biome check`,
   `npx expo-doctor`.

Commit atomically per logical step (scaffold â†’ metro/monorepo â†’ tokens/nativewind â†’
native deps â†’ smoke+verify), GSD style, on branch feat/mobile-scaffold.
</approach>

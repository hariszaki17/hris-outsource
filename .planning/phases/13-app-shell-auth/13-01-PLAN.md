<objective>
Build the mobile app shell: API/i18n/Query providers, an Expo Router auth gate, a login screen,
and the three READY-backend screens (Beranda dashboard, Notifications, More/logout). This is the
foundation every later agent screen sits on. Backend is READY (auth, dashboards/me, notifications
all implemented + agent-callable) — so this phase is almost entirely React Native.
</objective>

<context>
- Reuse @swp/api-client hooks (Orval). Hooks return the mutator envelope `{ data, status, headers }`
  — read `result.data`. configureApiClient({ baseUrl, getToken, onUnauthenticated }).
- RN auth diverges from web: web uses an httpOnly refresh COOKIE; RN cannot. Store refresh_token in
  expo-secure-store, send it in the BODY to POST /auth/refresh (AuthRefreshBody { refresh_token }).
  Access token stays in memory (auth module variable) and is returned by getToken().
- Hooks/types: useAuthLogin {data: LoginRequest{identifier,password,stay_signed_in?}} ->
  LoginResponse{access_token,refresh_token,expires_in,user:MeResponse}; useAuthRefresh
  {data:{refresh_token}} -> {access_token,refresh_token?}; useAuthLogout (void); useAuthMe ->
  MeResponse{id,role,full_name,employee_id,status,scope}. E10: useGetMyDashboard -> Dashboard
  (AgentDashboard variant); useListNotifications -> {notifications:Notification[]} + cursor;
  useGetNotificationCount -> {unread}; useMarkNotificationRead {notificationId};
  useMarkAllNotificationsRead {data:{before_timestamp?}} -> {count}.
- i18n: @swp/shared/i18n exports `resources`, `DEFAULT_LOCALE='id'`, `Locale`. Init i18next + react-i18next.
- Base URL: EXPO_PUBLIC_API_BASE_URL, default http://localhost:8081/api/v1 (BE dev port). Document that
  a real device needs the host LAN IP (localhost = the device itself).
- New deps: expo-secure-store (expo install), i18next + react-i18next + @tanstack/react-query (pnpm add).
- Replace the smoke screen (app/index.tsx) with the auth-gated entry.
</context>

<approach>
1. Deps: `npx expo install expo-secure-store`; `pnpm --filter @swp/mobile add i18next react-i18next @tanstack/react-query`.
2. src/lib/auth.ts — module token store: `accessToken` in memory; `setSession/clearSession/getRefresh`
   over SecureStore (key 'swp.refresh'); `getToken()`.
3. src/lib/api.ts — `configureApiClient({ baseUrl: EXPO_PUBLIC_API_BASE_URL ?? default, getToken,
   onUnauthenticated: () => sessionBus.signOut() })`. Imported once at startup.
4. src/lib/i18n.ts — i18next.init({ resources, lng: DEFAULT_LOCALE, fallbackLng: 'en' }).
5. src/providers/session.tsx — SessionProvider: status 'restoring'|'authed'|'unauthed', user, signIn
   (store tokens+user), signOut (useAuthLogout best-effort + clearSession). On mount: restore via
   customFetch('/auth/refresh',{POST, body:{refresh_token}}) then customFetch('/auth/me').
   Expose via useSession().
6. src/providers/app-providers.tsx — QueryClientProvider(createQueryClient) + I18nextProvider +
   SafeAreaProvider + SessionProvider.
7. app/_layout.tsx — wrap in AppProviders; run checkForJsUpdate() on launch; render <Slot/>.
8. app/index.tsx — redirect by session status (Redirect to /(app) or /(auth)/login; splash while restoring).
9. app/(auth)/_layout.tsx + login.tsx — react-hook-form-free simple controlled form (identifier +
   password), useAuthLogin -> session.signIn; error/loading/locked states; i18n copy.
10. app/(app)/_layout.tsx — Tabs: Beranda, Notifikasi, Lainnya (add Attendance/Schedule tabs in their phases).
11. app/(app)/index.tsx — Beranda: useGetMyDashboard; render agent dashboard summary; loading/empty/error.
12. app/(app)/notifications.tsx — list + unread count + mark-read + mark-all; states.
13. app/(app)/more.tsx — profile summary from useAuthMe, change-password stub, logout.
14. Add RN primitives as needed (Button, Card, ListItem) in src/ui backed by tokens.
</approach>

Hook → screen → real endpoint map (from frontend/packages/api-client/src/e3.ts, called by the screens):
  useListPlacements           → placements-screen.tsx           → GET  /placements
  useListExpiringPlacements   → placements-screen.tsx (toggle)  → GET  /placements?expiring_within_days=N
  useGetPlacement             → placement-detail-screen.tsx     → GET  /placements/{id}
  useCreatePlacement          → placement-form.tsx              → POST /placements           vars { data }
  useRenewPlacement           → placement-overlays RenewModal   → POST /placements/{id}:renew     vars { id, data }
  useTransferPlacement        → placement-overlays TransferModal→ POST /placements/{id}:transfer  vars { id, data }
  useEndPlacement             → placement-overlays EndConfirm   → POST /placements/{id}:end       vars { id, data }
  useResignPlacement          → placement-overlays ResignModal  → POST /placements/{id}:resign    vars { id, data }
  useTerminatePlacement       → placement-overlays TerminateConfirm → POST /placements/{id}:terminate vars { id, data }
  useGetCompanyRoster         → company-roster-screen.tsx       → GET  /client-companies/{id}/roster
  useCreateShiftLeaderAssignment   → ShiftLeaderAssignModal     → POST /shift-leader-assignments         vars { data }
  useReplaceShiftLeaderAssignment  → ShiftLeaderReplaceModal    → POST /shift-leader-assignments/{id}:replace vars { id, data }
  useEndShiftLeaderAssignment      → ShiftLeaderEndConfirm      → POST /shift-leader-assignments/{id}:end      vars { id, data }

Real response shapes the components destructure (BE MUST match these or the component breaks):
  List/expiring (`query.data?.data`): { data: Placement[], next_cursor?: string|null, has_more?: boolean }.
    Each Placement row reads: id, employee_id, employee_name, client_company_name, service_line_name,
    position_name, start_date, end_date (null = open-ended), lifecycle_status. (placements-screen columns)
  Detail (`query.data?.data` = PlacementDetailResponse): { placement: Placement, history_chain: PlacementSummary[],
    current_shift_leader: ShiftLeaderAssignmentSummary | null }. placement-detail-screen ALSO reads on `placement`:
    warnings[] (checks 'NO_SHIFT_LEADER_AT_COMPANY'), agreement_type, agreement_id, annual_leave_entitlement_days,
    base_salary_ref_idr, notes, created_by, created_at, status_changed_at, successor_id, termination_reason,
    resign_at, client_company_id. history_chain items (PlacementSummary) read: id, client_company_name,
    service_line_name, start_date, end_date, lifecycle_status. current_shift_leader reads: id, employee_id, employee_name.
  Roster (`query.data?.data` = CompanyRosterResponse): { company_name, placements: Placement[],
    current_shift_leader: {id,employee_id,employee_name}|null, summary: { total_active, total_scheduled,
    total_expiring, by_service_line: [{service_line_id, service_line_name}], by_status }, has_more, next_cursor }.
  INV envelope the form/overlays render: error.details = INVViolationDetails. placement-form reads
    details.current_placement.{client_company_name, client_company_id, lifecycle_status} and
    details.suggested_actions[] (enum: end, transfer, replace, end_existing_first, assign_after_placement),
    gated by ApiError.isInvariantViolation + code.startsWith('INV_1'). Overlays surface INV-2/3/4 via
    classifyError(err) kind 'invariant'|'conflict' → Banner(message).

Existing facts (verified from the codebase — use directly, do not re-explore):
- Frontend monorepo: pnpm@9.6.0 + Turborepo. `frontend/pnpm-workspace.yaml` currently lists only `apps/*` and `packages/*` (no `e2e`). Root `frontend/package.json` scripts use `turbo run …` and a `gen` script.
- Web app is `@swp/web` at `frontend/apps/web` (Vite 6 + React + TanStack Router). MSW toggled by `import.meta.env.VITE_ENABLE_MSW === 'true'` in `frontend/apps/web/src/main.tsx`; API base by `VITE_API_BASE_URL` (read in `frontend/apps/web/src/lib/auth.ts`).
- Backend: Go 1.23. `backend/cmd/migrate` runs goose (`go run ./cmd/migrate up`, reads `DATABASE_URL` via `config.Load`). River migrate: `river migrate-up --database-url <url>` (in `backend/Makefile` target `river-migrate`). Seed command `backend/cmd/seed` is delivered by plan 01-02 (invoke as `go run ./cmd/seed`).
- Go API entrypoint: `backend/cmd/api` (`go run ./cmd/api`); listens on `HTTP_ADDR` (default `:8080`), health endpoint `GET /healthz` returns 200 "ok". Required env: `DATABASE_URL`, `AUTH_JWT_PRIVATE_KEY`, `AUTH_JWT_PUBLIC_KEY` (base64 std Ed25519), `CORS_ALLOWED_ORIGINS`, `AUTH_COOKIE_SECURE`. Dev keypair generated by `go run ./cmd/seed -genkeys` (delivered by plan 01-02).
- Dev Postgres (docker-compose.yml) uses port 5432; the e2e Postgres MUST use 5433 to avoid clashing.

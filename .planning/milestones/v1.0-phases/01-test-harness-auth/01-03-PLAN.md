MeResponse (GET /auth/me 200, and the `user` field of LoginResponse):
```jsonc
{

}
```
scope rules: hr_admin/super_admin → {type:"global", company_id:null}; shift_leader → {type:"company", company_id:<users.company_id>}; agent → {type:"self", company_id:null}.

POST /auth/forgot-password: body {email}; ALWAYS 202 `{ "message": "..." }` if well-formed (anti-enumeration C-2); 400 on malformed.
POST /auth/reset-password: body {reset_token, new_password (minLength 10)}; 204 on success; 401 `RESET_TOKEN_EXPIRED` if token invalid/expired/used; 422 `WEAK_PASSWORD` with fields.new_password if policy fails. Optional Idempotency-Key parameter.
POST /auth/refresh: body {refresh_token} OR cookie → RefreshResponse; 401 UNAUTHENTICATED on invalid.
POST /auth/logout: 204; clears cookie + revokes refresh token.

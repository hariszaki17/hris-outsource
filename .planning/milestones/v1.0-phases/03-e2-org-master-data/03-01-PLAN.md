Cursor list pattern (from db/queries/identity/users.sql ListUsers) — keyset on (created_at,id):
```sql
WHERE deleted_at IS NULL
  AND (sqlc.narg(status)::text IS NULL OR status = sqlc.narg(status))
  AND (sqlc.narg(cursor_created_at)::timestamptz IS NULL
       OR (created_at, id) < (sqlc.narg(cursor_created_at)::timestamptz, sqlc.narg(cursor_id)::text))
ORDER BY created_at DESC, id DESC
LIMIT sqlc.arg(row_limit);
```
DB stores status lowercase ('active'/'inactive'); the DTO boundary uppercases to ACTIVE/INACTIVE (Phase-2 decision). Service lowercases incoming status filters before querying.

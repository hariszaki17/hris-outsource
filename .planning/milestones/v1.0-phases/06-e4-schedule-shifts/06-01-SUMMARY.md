## Reference for 06-02 (data-layer hand-off)

**Tables & key columns**

- `shift_masters` (id `SWP-SHF-*` via column DEFAULT): `name`, `start_time`/`end_time`/`break_start`/`break_end` = **text HH:MM**, `service_line_id` (nullable FK), `cross_midnight` (bool, server-derived — 06-02 sets it), `is_active`, `created_by`, `created_at`/`updated_at`, `deleted_at`.
- `schedule_entries` (id `SWP-SCH-*`): `employee_id` (FK), `placement_id` (FK, INV-2 anchor), `service_line_id` (nullable FK), `shift_master_id` (nullable FK — NULL only when `is_day_off`), `start_time`/`end_time` = **nullable text HH:MM snapshot**, `cross_midnight`, `work_date` (date), `status` CHECK(`SCHEDULED|CANCELLED_BY_LEAVE|MODIFIED`), `is_day_off`, `replaced_entry_id` (self-FK), audit cols, `deleted_at`.
- `approved_leave_days` (id `bigserial`): `employee_id` (FK), `leave_date` (date), `leave_request_id` (text, **no FK** — display only), `leave_type` (text), `created_at`.

**Indexes** (rely on these; don't add duplicates)

- `schedule_entries_active_agent_date_uq` — partial UNIQUE on `(employee_id, work_date) WHERE deleted_at IS NULL` → the DOUBLE_SHIFT backstop. 06-02 service does a `FindLiveEntryForAgentDate` pre-check, then catches the **pgx `23505`** unique-violation as the race-safe fallback (mirror Phase-5 INV-1 handling).
- `shift_masters_name_uq` — partial UNIQUE on `(name) WHERE deleted_at IS NULL` → a create/update name clash surfaces as `23505`; map to the SM-4 conflict.
- Non-unique helpers: `shift_masters_service_line_idx`, `schedule_entries_placement_idx`, `schedule_entries_company_range_idx` (on `work_date`), `schedule_entries_shift_master_idx`.
- `approved_leave_days_emp_date_uq` — UNIQUE `(employee_id, leave_date)` (seed one row per agent/date).

**Querier method names (sqlc, in `internal/repository/sqlc`)**

- Shift masters: `ListShiftMasters` (`:many`), `GetShiftMaster` (`:one`), `GetShiftMasterForUpdate` (`:one`, FOR UPDATE), `CreateShiftMaster` (`:one`), `UpdateShiftMaster` (`:one`), `SetShiftMasterActive` (`:one` — drives `:deactivate`/`:reactivate`).
- Schedule: `ListSchedule` (`:many`), `GetScheduleEntry` (`:one`), `GetScheduleEntryForUpdate` (`:one`, FOR UPDATE), `FindLiveEntryForAgentDate` (`:one` — DOUBLE_SHIFT pre-check / replace lookup), `CreateScheduleEntry` (`:one`), `UpdateScheduleEntry` (`:one`), `SoftDeleteScheduleEntry` (`:execrows`), `FindActivePlacementForAgentDate` (`:one` — INV-2 / OUTSIDE_PLACEMENT_PERIOD).
- Over-leave: `FindApprovedLeaveForAgentDate` (`:one` — SHIFT_OVER_LEAVE; returns `leave_request_id`, `leave_type`, `leave_date`).

**sqlc type quirks (handle these in the repository mapping)**

- **`pgtype.Date`, not `time.Time`** for all `date` columns: `ListScheduleRow.WorkDate`, `ListScheduleParams.StartDate/EndDate`, `FindActivePlacementForAgentDateRow.StartDate/EndDate`, the leave row's `LeaveDate`. Convert at the repo boundary exactly like Phase-5 placements: read `.Time`/`.Valid`; write `pgtype.Date{Time: t, Valid: true}`.
- **`in_use_count` → `int64`** (`InUseCount int64` on `ListShiftMastersRow`/`GetShiftMasterRow`) — matches `domain.ShiftMaster.InUseCount int64`.
- **Nullable text → `*string`** (emit_pointers_for_null_types): `BreakStart`, `BreakEnd`, `ServiceLineID`, `ShiftMasterID`, `StartTime`, `EndTime`, `ReplacedEntryID`, `ServiceLineName`, `EmployeeName`, `CompanyName`, `ShiftMasterName`, `CreatedBy`. `CompanyID` on schedule rows is **non-null `string`** (it comes from the `JOIN placements`).
- **`status_in` filter is `[]string`** (`ListScheduleParams.StatusIn`); the query is `(narg::text[] IS NULL OR status = ANY(...))` — pass `nil` to skip.
- **`is_active` filter (`*bool`) maps the API ACTIVE/INACTIVE token**: 06-02 translates `ShiftMasterFilter.Status` → `*bool` (ACTIVE→true, INACTIVE→false, unset→nil).
- `ListShiftMasters` cursor is **id-desc keyset** (`CursorID *string`, `RowLimit int32`); `ListSchedule` is **not paginated** (full company×date-range window, ordered `employee_id, work_date, id`).
- `CreateScheduleEntry` / `CreateShiftMaster` **omit `id`** (column DEFAULT fires); `cross_midnight` is passed in (service derives `end<=start`).

**Domain types ready for the DTO layer**

- `domain.ShiftMaster`, `domain.ScheduleEntry` (`WorkDate time.Time`), `domain.ShiftMasterFilter`, `domain.ScheduleFilter` in `internal/domain/scheduling.go`. Reuse `domain.ErrNotFound` (already mapped from `pgx.ErrNoRows` in the repo convention). The openapi DTO field list 06-02 must satisfy is inlined in the plan's `<context>` block.

  - Loop ids; each acted in its OWN attempt; success → append id to Succeeded; failure →
    map apperr via `apperr.As(err)` to a Failed row {id, error{code,message}}; never abort the loop

  to match openapi BulkActionResponse (NOT the scheduling CellResult shape).

Scope guard (schedule_service.go line ~100, 322): after loading the record's company_id,
  `if serr := rbac.GuardCompany(ctx, rec.CompanyID); serr != nil { return ... OUT_OF_SCOPE ... }`.
  GuardCompany returns nil for hr_admin/super_admin (global), and an error for a shift_leader whose
  assignment company != rec.CompanyID. Map that to apperr code "OUT_OF_SCOPE" HTTPStatus 403.

apperr usage:
  - `apperr.Conflict("CODE")` → 409 (terminal-state, CORRECTION_ALREADY_PENDING, IDEMPOTENCY_KEY_REUSED).
  - `apperr.Invalid(map[string]string{...})` → 422 with fields.
  - Non-default status uses STRUCT LITERAL: `&apperr.Error{Code:"VERIFY_OWN_RECORD", HTTPStatus:403, Message:...}`
    and `&apperr.Error{Code:"OUT_OF_SCOPE", HTTPStatus:403, ...}`,
    `&apperr.Error{Code:"OUTSIDE_CORRECTION_WINDOW", HTTPStatus:422, Fields: map[string]string{"attendance_date":..,"window_days":"7"}}`.
    (Precedent: GEOFENCE_RADIUS_INVALID/FILE_TOO_LARGE struct literals, decisions [03/04].)
  - `apperr.As(err)` unwraps to read .Code/.HTTPStatus/.Message/.Fields for the bulk Failed rows.

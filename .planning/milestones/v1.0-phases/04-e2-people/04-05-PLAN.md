From backend/internal/handler/org/companies_handler_test.go:
- package `people_test`.
- `fakeTx` implementing pgx.Tx with only Exec as no-op (others panic) — copy verbatim;
  audit.Record runs inside InTx closures, so the fake TxRunner must pass a *fakeTx.
- `fakeTxRunner` implementing `InTx(ctx, fn) error` → `return fn(&fakeTx{})`.
- in-memory fake repos implementing the service's consumer interfaces (slices/maps as backing store).
- a chi router built with the same rbac groups + a middleware that injects a configurable
  principal (auth.WithPrincipal) so role/scope can be swapped per test.
- assertion style: decode response body into map[string]any (or a typed struct) and assert
  exact keys + values; assert resp.StatusCode.

Codes/shapes to assert (authoritative = openapi.yaml examples):
- Employee: snake_case keys incl. bank_account{bank_name,account_number,account_holder_name},
  status UPPERCASE, has_login bool.
- DUPLICATE_NIK → 409 with error.fields.nik.
- Agreement: end_date nullable; status UPPERCASE incl. EXPIRING virtual; compensation.bpjs_terms 4 pcts.
- PKWT no end_date → 400 INVALID_REQUEST; PKWT > 5y → 422 PKWT_PERIOD_EXCEEDS_MAX (error.fields.end_date);
  2nd active → 409 ACTIVE_AGREEMENT_EXISTS.
- FileRef from upload: {id, url, name, size_bytes, mime, uploaded_at} — assert all 6 keys; 413 on oversize.
- ChangeRequest detail: diff{<field>:{old,new}}; approve → 200 status APPROVED; reject no reason → 400;
  reject ok → 200 status REJECTED + rejection_reason; approve/reject on resolved → 409 CONFLICT.
- RBAC: a shift_leader/agent calling a write endpoint → 403 (where the route group excludes them).

    `text PRIMARY KEY` ids, `created_at/updated_at timestamptz NOT NULL DEFAULT now()`,
    and `deleted_at timestamptz` for soft-delete.

    **00016_employees.sql** — CREATE TABLE employees:
      - id text PRIMARY KEY (inserts build 'SWP-EMP-' || swp_next_id('EMP'))
      - user_id text (nullable; linked E1 user)
      - full_name text NOT NULL
      - nik text NOT NULL
      - nip text NOT NULL DEFAULT ''
      - join_at date NOT NULL
      - gender text (nullable; CHECK gender IN ('MALE','FEMALE') OR gender IS NULL)
      - birth_date date (nullable), birth_place text (nullable)
      - phone text (nullable), email_personal text (nullable), address text (nullable)
      - npwp text (nullable), bpjs_kesehatan text (nullable), bpjs_ketenagakerjaan text (nullable)
      - bank_name text, bank_account_number text, bank_account_holder_name text (all nullable — flat columns for the bank_account object)
      - status text NOT NULL DEFAULT 'active' CHECK (status IN ('active','inactive'))
      - created_by text (nullable)
      Partial unique index `employees_nik_uq ON employees (nik) WHERE deleted_at IS NULL`.
      No FK on user_id (E1 users table is preserved separately; keep loose like Phase-1 persona literals).
      Down: DROP TABLE IF EXISTS employees.

    **00017_employment_agreements.sql** — CREATE TABLE employment_agreements:
      - id text PRIMARY KEY ('SWP-AG-' || swp_next_id('AG'))
      - employee_id text NOT NULL REFERENCES employees(id)
      - type text NOT NULL CHECK (type IN ('PKWT','PKWTT'))
      - agreement_no text NOT NULL DEFAULT ''
      - start_date date NOT NULL
      - end_date date (nullable; required for PKWT, enforced in service)
      - status text NOT NULL DEFAULT 'active' CHECK (status IN ('active','superseded','closed'))
      - predecessor_id text (nullable), successor_id text (nullable)
      - closed_reason text (nullable; CHECK closed_reason IN ('RESIGNED','TERMINATED','END_OF_TERM','OTHER') OR closed_reason IS NULL)
      - closed_at timestamptz (nullable)
      - base_salary_idr numeric (nullable), bpjs_terms jsonb (nullable), tax_profile text (nullable), comp_effective_date date (nullable)
        (compensation stored as plain columns this milestone — note "encryption at rest deferred" in SUMMARY)
      - created_by text (nullable)
      Partial unique index `employment_agreements_active_employee_uq ON employment_agreements (employee_id) WHERE status = 'active' AND deleted_at IS NULL` (enforces EA-2: one active agreement per employee at the DB level).
      Down: DROP TABLE IF EXISTS employment_agreements.

    **00018_agreement_attachments.sql** — CREATE TABLE agreement_attachments:
      - id text PRIMARY KEY ('SWP-FILE-' || swp_next_id('FILE'))
      - agreement_id text NOT NULL REFERENCES employment_agreements(id)
      - category text NOT NULL DEFAULT 'signed_agreement' CHECK (category IN ('signed_agreement','addendum','supporting_doc'))
      - caption text NOT NULL DEFAULT ''
      - file_name text NOT NULL
      - mime text NOT NULL
      - size_bytes bigint NOT NULL
      - blob bytea NOT NULL  (storage choice: bytea in-DB — simplest that passes E2E + survives container teardown via reseed; document in SUMMARY)
      - uploaded_by text (nullable)
      Down: DROP TABLE IF EXISTS agreement_attachments.

    **00019_change_requests.sql** — CREATE TABLE change_requests:
      - id text PRIMARY KEY ('SWP-CHG-' || swp_next_id('CHG'))
      - employee_id text NOT NULL REFERENCES employees(id)
      - status text NOT NULL DEFAULT 'pending' CHECK (status IN ('pending','approved','rejected'))
      - changes jsonb NOT NULL  (the {phone?,address?,bank_account?} subset)
      - request_type text NOT NULL CHECK (request_type IN ('PHONE','ADDRESS','BANK_ACCOUNT','MULTIPLE'))
      - note text (nullable)
      - submitted_at timestamptz NOT NULL DEFAULT now()
      - resolved_at timestamptz (nullable), resolved_by text (nullable), rejection_reason text (nullable)
      Index `change_requests_status_idx ON change_requests (status, submitted_at DESC, id DESC)`.
      Down: DROP TABLE IF EXISTS change_requests.

    Status values are stored lowercase in DB and uppercased to ACTIVE/PKWT/PENDING etc.
    at the DTO boundary (matches the Phase-2 decision in STATE.md: "DB lowercase
    'active'/'disabled' uppercased ... only at DTO boundary").

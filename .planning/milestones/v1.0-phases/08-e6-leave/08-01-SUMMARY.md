# Phase 8 Plan 01: E6 Leave Data Layer Summary

Stood up the E6 leave **data layer** — three goose migrations (`leave_requests`,
`leave_quotas`, `leave_approvals`), the `backend/db/queries/leave/` sqlc query set
covering every read/write 08-02 needs, and the `internal/domain/leave` type package
— with the openapi-authoritative `LeaveStatus` enum and the soft-reservation quota
balance model pinned at the schema so they cannot drift. `make gen` + `go build` +
`go vet` clean; no services, routes, or HTTP (wave-1 foundation only).

## What was built

**Task 1 — Migrations 00028-00030** (commit `d583e14`)
- `00028_leave_requests`: `SWP-LR` id via column DEFAULT, status CHECK
  (`DRAFT/PENDING_L1/PENDING_HR/APPROVED/REJECTED/CANCELLED`), denormalized
  `company_id`/`service_line_id` for leader-scope queue + calendar, `routing.*`
  (`no_leader`, `assigned_leader_id`) and `balance_check.*` snapshot columns, the
  `(company_id,status)` + `(employee_id)` partial indexes, soft-delete.
- `00029_leave_quotas`: `SWP-LQ` id, `period`/`period_start`/`period_end`, the
  `total`/`used`/`pending` soft-reservation triple (`remaining` derived, not stored),
  `is_prorated`/`prorate_months`/`closed`, `last_adjustment`/`last_override` jsonb,
  unique `(employee_id,leave_type_id,period)`.
- `00030_leave_approvals`: bigserial decision-trail (`stage` L1/HR, `decision`
  APPROVED/REJECTED/OVERRIDE_APPROVED, `is_override`+reason, actor), request+occurred_at index.

**Task 2 — sqlc queries + make gen + domain types** (commit `55f8319`)
- 4 query files → regenerated `internal/repository/sqlc`; `internal/domain/leave/leave.go`
  with `LeaveQuota.Remaining()` and the full openapi surface.

## Decisions made (Claude's Discretion items from CONTEXT)

1. **leave_approvals = separate table** (not denormalized columns): a bigserial
   append-only decision row per action mirrors `placement_history`; cleanly feeds the
   FE `timeline[]` and the FEATURE ER decision log.
2. **Soft-reservation recompute = computed-on-read**: `CountPendingLeaveDaysForQuota`
   sums open `PENDING_L1`/`PENDING_HR` request days for the (employee,type,period) —
   simplest correct option; no triggers, no drift between writers.
3. **SWP-LR id reconciliation**: documented in the 00028 header — the new entity ids
   share the `swp_next_id('LR')` sequence with Phase-6's literal `SWP-LR-44210` fixture
   without collision; 08-02 writes the real created id into `approved_leave_days`.

### sqlc Querier methods (package `sqlcgen`, interface `Querier`)

leave_requests.sql:
- `ListLeaveRequests(ctx, ListLeaveRequestsParams) ([]ListLeaveRequestsRow, error)` — keyset cursor; filters all `*string`/`[]string`/`pgtype.Date` nargs; `Lim int32`.
- `GetLeaveRequest(ctx, id string) (GetLeaveRequestRow, error)` — denormalized (incl. `LeaveTypeCode`).
- `GetLeaveRequestForUpdate(ctx, id string) (GetLeaveRequestForUpdateRow, error)` — FOR UPDATE, no joins.
- `CreateLeaveRequest(ctx, CreateLeaveRequestParams) (CreateLeaveRequestRow, error)` — id via DEFAULT.
- `CreateLeaveRequestWithID(ctx, CreateLeaveRequestWithIDParams) (..., error)` — explicit id, `ON CONFLICT (id) DO NOTHING` (use for **seed** deterministic targets, e.g. SWP-LR for E2E).
- `UpdateLeaveRequestStatus(ctx, UpdateLeaveRequestStatusParams) (UpdateLeaveRequestStatusRow, error)` — the state transitions; also writes routing + balance_check snapshot cols.
- `CountPendingLeaveDaysForQuota(ctx, CountPendingLeaveDaysForQuotaParams) (int64, error)` — `pending_days bigint` (drives quota.pending on read).

leave_quotas.sql:
- `ListLeaveQuotas`, `GetLeaveQuota`, `GetLeaveQuotaForUpdate (FOR UPDATE)`, `FindQuotaForEmployeeTypePeriod` — all return the `LeaveQuota`/`*Row` shape.
- `UpsertLeaveQuota(ctx, UpsertLeaveQuotaParams) (LeaveQuota, error)` — `ON CONFLICT (employee_id,leave_type_id,period)`; updates `total`/`is_prorated`/`prorate_months` only (does **not** touch `used`/`pending`).
- `AdjustLeaveQuotaTotal(ctx, AdjustLeaveQuotaTotalParams) (LeaveQuota, error)` — `total = total + Delta`, sets `last_adjustment []byte`. **Service must pre-check** `total+delta >= used` → 422 `RULE_VIOLATION` (query does not guard).
- `DeductLeaveQuota` (used += delta) / `RestoreLeaveQuota` (used = GREATEST(used-delta,0)) / `SetLeaveQuotaOverride` (last_override jsonb).
- `ListActivePlacedEmployeesForGrant(ctx, ListActivePlacedEmployeesForGrantParams) ([]...Row, error)` — employees with ACTIVE/EXPIRING placement covering the period (the `employee_ids:["all"]` sentinel + pro-rate join-date source; row carries `EmployeeID`, `EmployeeName *string`, `PlacementStartDate pgtype.Date`).

leave_approvals.sql:
- `InsertLeaveApproval(ctx, InsertLeaveApprovalParams) (LeaveApproval, error)`.
- `ListLeaveApprovalsForRequest(ctx, leaveRequestID string) ([]LeaveApproval, error)` — timeline source, ORDER BY occurred_at.

leave_calendar.sql:
- `ListCalendarEntries(ctx, ListCalendarEntriesParams) ([]ListCalendarEntriesRow, error)` — params: `RangeTo`/`RangeFrom pgtype.Date`, `StatusIn []string` (**service builds it**: `["APPROVED"]` when `show_pending=false`, else `["APPROVED","PENDING_L1","PENDING_HR"]`), `CompanyID`/`ServiceLine`/`LeaveTypeID *string` nargs.

### sqlc type quirks (repo must convert — mirrors Phase-5/6/7)

- **Dates** (`start_date`/`end_date`/`period_start`/`period_end`) → `pgtype.Date`. Convert `<-> time.Time` (`.Time` field + `Valid`); same pattern as the scheduling repo.
- **jsonb** (`last_adjustment`/`last_override`) → `[]byte`. Marshal/unmarshal `domain.LeaveQuotaAdjustment` / `LeaveQuotaOverride` (json). nil `[]byte` → nil pointer.
- **ints** → `int32` in sqlc (`Period`, `Total`, `Used`, `Pending`, `DurationDays`, `ProrateMonths`, `BalanceRequestedDays *int32`, …) — domain uses plain `int`; cast at the boundary.
- **CountPendingLeaveDaysForQuota** returns `int64` (bigint).
- **leave_approvals.ID** is `int64` (bigserial), not a string id.
- Nullable columns → pointers (`*string`/`*int32`/`*bool`); `pgx.ErrNoRows → domain.ErrNotFound` in the repo (08-02).
- `Status` comes back as plain `string`; cast to `domain/leave.LeaveStatus`.

### INV-3 loop-closer queries — **08-02 MUST ADD** (do not exist yet)

These two belong in `backend/db/queries/scheduling/` (the E4-owned dir, alongside `approved_leave_days.sql` / `schedule_entries.sql`) so the loop-closer writes into the E4 tables. Add them, `make gen`, then call inside the approve-final/override tx:

```sql
-- in backend/db/queries/scheduling/approved_leave_days.sql
-- name: InsertApprovedLeaveDay :exec
-- INV-3 write-through: the REAL leave_requests.id replaces the Phase-6 fixture.
INSERT INTO approved_leave_days (employee_id, leave_date, leave_request_id, leave_type)
VALUES (sqlc.arg(employee_id), sqlc.arg(leave_date), sqlc.arg(leave_request_id), sqlc.narg(leave_type))
ON CONFLICT (employee_id, leave_date) DO UPDATE
  SET leave_request_id = EXCLUDED.leave_request_id, leave_type = EXCLUDED.leave_type;

-- in backend/db/queries/scheduling/schedule_entries.sql
-- name: CancelScheduleEntriesForLeave :many
-- INV-3: cancel overlapping live schedule entries for the leave dates; RETURNING
-- feeds LeaveRequest.schedule_impact[] (prior_status/new_status/clock_in_conflict).
UPDATE schedule_entries
SET status = 'CANCELLED_BY_LEAVE', updated_at = now()
WHERE employee_id = sqlc.arg(employee_id)
  AND work_date BETWEEN sqlc.arg(start_date)::date AND sqlc.arg(end_date)::date
  AND status <> 'CANCELLED_BY_LEAVE'
  AND deleted_at IS NULL
RETURNING id, work_date, status;
```

Notes for 08-02 wiring:
- `approved_leave_days` has unique `(employee_id, leave_date)` — the `ON CONFLICT` upsert is required (a re-approve / overlapping day must not 23505).
- `schedule_entries.status` CHECK is `('SCHEDULED','CANCELLED_BY_LEAVE','MODIFIED')` — use `CANCELLED_BY_LEAVE` (NOT a `LEAVE`/`UNASSIGNED` literal; the domain `ScheduleImpactEntry.NewStatus` maps to the FE's `LEAVE` label at the DTO boundary).
- `approved_leave_days.leave_request_id` has **no FK** — insert the new `leave_requests.id` string directly; this is the "E6 wires the production source" hand-off Phase-6 documented.
- All of the above runs in the same `db.TxManager.InTx` as `UpdateLeaveRequestStatus` + `DeductLeaveQuota` + `InsertLeaveapproval` + `audit.Record` + the notify stub.

<decisions_made>

- **Route location**: `POST /attendance:manual-create` in existing admin/leader group (same scope as verify/reject), not a separate group ‚Äî HR/super_admin only, shared idempotency wrapper
- **No dedicated PRD doc**: Feature documented inline, not as separate PRD. Business rules MR-1 through MR-11 defined in workflow prompt
- **Flag strategy**: Always `MANUAL_ENTRY` + lateness/early flags when applicable + `PENDING` verification. No `AUTO_APPROVED` path for manual entries ‚Äî HR fills in; leader/another HR must verify
- **Geofence**: Bypassed entirely for manual entries. `lat_in/lng_in = null`, `in_geofence = true`, `in_distance_m = 0`
- **Placement resolution**: Reused `GetActivePlacement` + `GetTodaySchedule` from ClockRepo pattern but added directly to `AttendanceRepository` interface (not shared via ClockRepository) to avoid circular dependency
- **Time validation**: check_out_at must be >= check_in_at (422 if violated). RFC3339 format required

</decisions_made>

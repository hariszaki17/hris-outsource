# Web Stack — hris-outsource

> The locked technology stack for the **web console** (Super Admin · HR/Placement Admin · Shift Leader).
> Companion to [ENGINEERING.md](ENGINEERING.md) (how we build) — this doc is **what we build with**.
>
> **Status:** v1 locked 2026-06-03. **Owner of changes:** product+architecture; coordinate edits with engineering.
> ✅ = explicitly chosen · *(default)* = sensible default, overridable.

---

## 1. Context that drives these choices

The web console is an **internal-only, authenticated, data-dense operations tool** ([EPICS.md](../EPICS.md) §2) talking to a **separate Go API** whose contract is already written ([docs/api/](../api/), OpenAPI 3.1 per epic). There is **no public surface, no SEO, no marketing pages**. A second React Native surface (mobile, for agents + leaders) shares types, tokens, and the API client. These facts make SSR/RSC and a Node BFF low-value, and make code reuse across surfaces high-value.

## 2. Locked stack

| Layer | Choice | Marker |
|---|---|---|
| Runtime | **Vite + React 19 + TypeScript** — SPA, static bundle, no Node runtime in prod | ✅ |
| Repo | **Single monorepo** (Go + JS in one tree) | ✅ |
| JS workspace | **pnpm + Turborepo** under `frontend/` | *(default)* |
| API client | **Orval** → typed TanStack Query hooks + Zod schemas + MSW mocks, generated from `docs/api/*/openapi.yaml` | ✅ |
| Server state | **TanStack Query** | ✅ |
| Routing | **TanStack Router** (typed routes + typed search params) | ✅ |
| Forms | **React Hook Form + Zod** (Zod reused from Orval) | ✅ |
| UI | **Tailwind CSS + shadcn/ui**, theme = [DESIGN-SYSTEM.md](../design/DESIGN-SYSTEM.md) tokens | ✅ |
| Local UI state | **Zustand** for the little cross-component UI state; Context for truly static config | *(default)* |
| i18n | **react-i18next** — Bahasa Indonesia default, `en-US` via `Accept-Language` | *(default)* |
| Date/time | TZ-aware layer pinned to **`Asia/Jakarta`** (canonical per CONVENTIONS §10) | ✅ |
| Lint + format | **Biome** (single Rust tool, replaces ESLint+Prettier) | ✅ |
| Unit/component tests | **Vitest** + Testing Library | ✅ |
| E2E tests | **Playwright** | ✅ |
| API mocking | **MSW** (handlers generated by Orval from spec examples) | ✅ |
| Auth token model | **Deferred — locked when we design the Go `/auth` endpoints (E1)** | ⏳ |

## 3. Repository layout

```
repo/
  backend/                  Go API (built later)
  frontend/                 pnpm + turborepo workspace
    apps/
      web/                  Vite + React SPA  ← this doc
        src/
          features/<epic>/  mirrors docs/epics/<E#-name>/ (e.g. features/e6-leave/)
          app/              router, providers, shell, error boundary
          lib/              auth abstraction, api error mapper, i18n, tz
      mobile/               React Native (agents + leaders)
    packages/
      api-client/           Orval output (generated) + x-rbac / x-design-screens maps
      design-tokens/        DESIGN-SYSTEM.md tokens → Tailwind theme + RN
      ui/                   shadcn primitives, owned & themed
      shared/               branded ID types, utils, i18n catalogs, tz helpers
  docs/                     existing specs (source of truth)
```

**Rule:** `apps/web/src/features/<epic>/` mirrors the epic/PRD tree so code is navigable straight from the docs.

## 4. Codegen pipeline (Orval)

`docs/api/*/openapi.yaml` → Orval (CI + `pnpm gen`) → `packages/api-client`:

- **Typed Query/Mutation hooks** per operation (one module per epic spec, mirroring the file layout).
- **Zod schemas** — reused directly by React Hook Form so form validation == API contract.
- **MSW handlers** seeded from the spec's named `*Example` schemas → the web app is fully buildable and testable **before the Go API exists**.
- **Derived maps** (post-gen step): `x-rbac` → a client permission map (UI gating by role + scope); `x-design-screens` → an endpoint ↔ `.pen` frame traceability index.

**Hard rules:**
- Generated files are **never hand-edited** (E2 in ENGINEERING.md). Need a change → change the spec, regenerate.
- Spec and client are reconciled in CI; **contract drift fails the build**.

**Known caveat (2026-06-03):** 6 specs use `oneOf` and 2 use `discriminator`. Orval's OpenAPI 3.1 union → Zod output on those can be imperfect. Mitigation: hand-author those few union types in `packages/shared` or add a post-gen patch step; everything else generates cleanly. If union fidelity ever outweighs the MSW/Zod benefit, `@hey-api/openapi-ts` is the documented fallback.

## 5. Why not the alternatives (so we don't relitigate)

- **Not Next.js / Remix / TanStack Start** — they add a Node server + SSR for value (SEO, public pages, BFF) this internal tool doesn't need; Go is already the backend.
- **Not a JS-only or split-repo layout** — a single monorepo gives atomic cross-stack commits and lets web + mobile share `api-client`, `design-tokens`, and `shared` without versioned-package release overhead.
- **Not MUI/Ant** — a heavy vendor theme fights the bespoke tokenized design system; shadcn is copy-in and fully owned.
- **Not hand-written API types** — the OpenAPI contracts already exist and are rich; generating keeps them the single source of truth.

## 6. Deferred / open

- **Auth token model** (in-memory access + httpOnly refresh cookie vs other) — locked alongside the Go `/auth` design in E1. Web keeps a thin auth abstraction so either fits.
- **Deploy target for the static bundle** (CDN vs served by the Go binary) — decided at infra phase.
- **i18n library** — `react-i18next` is the default; revisit vs LinguiJS if extraction DX matters more.

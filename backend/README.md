# hris-outsource — backend (Go API)

Internal-only HRIS API for SWP. Implements the contract in [`docs/api/`](../docs/api/)
(REST/JSON, OpenAPI 3.1). See [`docs/eng/`](../docs/eng/) for the binding engineering
rules and [`docs/EPICS.md`](../docs/EPICS.md) for product decisions.

## Stack

| Concern | Choice |
|---|---|
| Language / router | Go 1.23 · chi |
| DB access | sqlc (type-safe SQL) + pgx/v5 |
| Migrations | goose (embedded in `cmd/migrate`) |
| HTTP handlers | hand-written against the OpenAPI contract (no server codegen — see "Contract" below) |
| Auth | EdDSA (Ed25519) JWT access + rotating opaque refresh token (hashed in PG) |
| Async / cron | River (Postgres-backed queue — no Redis) |
| Observability | slog + OpenTelemetry traces (OTLP) + Prometheus metrics (`/metrics`) |
| Config | env (caarlos0/env) + `.env` in dev |

## Layout

```
cmd/{api,worker,migrate}     entrypoints
internal/
  platform/                  cross-cutting kernel (the shared spine)
    config log obs db        env · slog · otel · pgx pool + TxManager
    apperr httpx             domain errors → ErrorEnvelope · request-id, cursor, cors, ratelimit
    auth rbac                JWT + refresh + authn middleware · roles/scope guards
    audit idempotency        tx-scoped audit writer · Idempotency-Key store
    ids i18n jobs            SWP-<ENTITY>-N formatter · Bahasa error msgs · River
  domain/                    dependency-free business types
  handler/<epic>/            HTTP boundary  (E1 identity is the reference slice)
  service/<epic>/            business logic (INV-*/BR-* live here)
  repository/<epic>/         sqlc-backed data access
  repository/sqlc/           GENERATED by sqlc — never hand-edit
db/
  migrations/                goose .sql (also sqlc's schema source)
  queries/<epic>/            sqlc .sql query files
```

Layers are sub-packaged by epic so code maps 1:1 to `docs/epics/`.

## First-time setup

```bash
cp .env.example .env                 # then generate dev keys (below) into it
make tools                           # sqlc, goose, river, gofumpt
docker compose up -d postgres
make gen                             # sqlc only
make migrate-up                      # app schema
make river-migrate                   # job-queue tables
make run                             # api on :8080  (make worker in another shell)
```

Generate dev Ed25519 keys (base64) for `AUTH_JWT_PRIVATE_KEY` / `AUTH_JWT_PUBLIC_KEY`:

```bash
go run ./... # or a small throwaway calling auth.GenerateKeypair()
```

(A `genkeys` subcommand is a TODO; `auth.GenerateKeypair()` exists for it.)

## Reference slice: E1 auth

`POST /api/v1/auth/login · /auth/refresh · /auth/logout` and `GET /auth/me`.
Two refresh-token transports off the same endpoints (header `X-Auth-Transport`):
- **cookie** (web SPA, default): httpOnly, `SameSite=Lax`, `Domain=.swp.example.com`
- **bearer** (mobile): refresh token in the JSON body, stored in secure storage

Refresh tokens rotate on every refresh; a reused (already-rotated) token revokes
the whole family (reuse detection). Only SHA-256 hashes are stored.

## Contract (no server codegen)

`docs/api/*/openapi.yaml` (OpenAPI **3.1**) is the contract. The **frontend** generates
its typed client from it via Orval. The **backend does not use server-side codegen** —
handlers are hand-written against the contract (the E1 auth slice is the reference).

Why: oapi-codegen doesn't support OpenAPI 3.1 (it fails on 3.1's `type: [string, null]`
nullables and `oneOf` on 5 of 9 specs). Rather than add a 3.1-specific generator or a
lossy down-convert step, the server is hand-written and kept faithful to the contract by
**contract tests**. The specs keep their full richness (`x-rbac`, `x-design-screens`,
examples) for the frontend pipeline and as the human contract.

For interactive API docs, point any Swagger UI / Scalar / Redoc at the `.yaml` files
(e.g. `npx @scalar/cli serve ../docs/api/E2-identity/openapi.yaml`) — no backend coupling.

## Conventions honored (where they live in code)

- Error envelope + status mapping → `platform/apperr` + `platform/httpx`
- Cursor pagination → `platform/httpx/cursor.go`
- Audit on every write (tx-atomic) → `platform/audit`
- Idempotency-Key (24h) → `platform/idempotency`
- RBAC roles + company/self scope → `platform/rbac`
- SWP IDs → `platform/ids` + `swp_next_id()` in migration 00001
- Rate limit (600/min) → `platform/httpx/ratelimit.go` (in-memory; per-instance — see note in file)
